Multithreaded-Simple-Python-Server

This website is currently running on an experimental, minimilistic yet multi-threaded web server written in python.

While getting my hands dirty with socket programming for the first time, I started off with a simple web server. This server can serve all the GET requests received at the port it is listening at. This is real good to work with the static websites which have substantially less number of requests per page to travel through the network. Even in case of concurrent requests this server can fairly perform by serving all the queued requests.

Watching this simple code working fine intrigued me to include threads in the same code. The performance of the same old server increased hugely. I decide to enhance the same code to write a dedicated server to host my personal website mansi.im. The new server is extended with the following features:

  • multi-threaded
  • logs all the incoming requests in a log file pythonServer.log
  • checks for directory traversal
  • prepends an HTML message in every response which specifies the time taken to generate each response in Âµs

Log file

The log file is created using log_requests(r, c). The incoming requests are logged in a file named pythonServer.log in the following format:

[Date and Time] [Request sent by the client] [IP address of client] [User-Agent]
